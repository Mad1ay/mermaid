# Termi-Odoo Implementation Project

## Огляд проєкту

Цей репозиторій містить документацію та ресурси для впровадження Odoo в компанії Termi.

## Структура проєкту

```
termi-odoo-implementation/
├── README.md                # Головний опис проєкту (для стейкхолдерів)
├── docs/                    # Документація
│   ├── business_requirements/
│   │   ├── brd_termi.md     # Business Requirements Document
│   │   └── gap_analysis.md  # Gap Analysis (аналіз розривів)
│   ├── use_cases/
│   │   └── use_cases.md     # Use Cases для Termi BRD
│   └── workflows/           # Згенеровані діаграми робочих процесів
└── src/                     # Скрипти автоматизації (на майбутнє)
```

## Документація

### Business Requirements
- **BRD (Business Requirements Document)**: Детальний опис бізнес-вимог для впровадження Odoo
- **Gap Analysis**: Аналіз розривів між поточними процесами та функціоналом Odoo

### Use Cases
Детальні сценарії використання системи для різних бізнес-процесів компанії Termi.

### Workflows
Візуальні діаграми робочих процесів, створені за допомогою Mermaid.

## Як використовувати

1. Ознайомтеся з документацією в папці `docs/`
2. Перегляньте діаграми робочих процесів у `docs/workflows/`
3. Використовуйте BRD та Use Cases як основу для налаштування Odoo

## Внесок змін

Всі зміни в документацію вносяться через pull requests на відповідну гілку проєкту.
